# SAP Development Partner — User Guide

**Version:** 2026-05-29  
**Operator:** Catchy Lab

This guide reflects the current deployment. Fees, AI rates, and commissions follow on-screen notices, admin announcements, and the Terms of Service.

In **English** UI mode, the top navigation uses short labels (**Console**, **Develop**, **Improve**, **Connect**). This guide uses the full service names below.


## 1. Overview

SAP Development Partner is a hub for **New ABAP Development (RFP)**, **Analyze & Improvement**, and **SAP-Connected Development** requests. AI assists with **interview → Development Proposal**. Consultant members may provide **offers**, **matching**, **FS**, **delivered code**, **as-built deliverables**, and **project settlement**.

- **AI credits:** Prepaid balance for AI interview, proposal, FS, code, and AI inquiry chat.
- **Project settlement:** Agreed development fees between requester and consultant after matching (separate from AI credits).
- **Language:** Header **KO / EN** toggle.


## 2. Getting started

### 2.1 Sign up and login

1. Register with email, password, name; agree to Terms and Privacy Policy
2. Choose **general member** or apply as **consultant** (limited menus until approved)
3. Complete email/mobile verification when required
4. Login and password recovery

### 2.2 AI credits

- Open **Usage & balance** (or similar) for balance, usage, and top-up history
- **Bank transfer:** submit a claim; balance updates after the Company confirms deposit
- **Online payment:** via PortOne or similar (card/PayPal where offered)
- Promotional credits may apply on sign-up or events

## 4. New ABAP Development (RFP)

### 4.1 Create a request

1. Open the **Develop** tile on the home page (service: **New ABAP Development**) or the matching top-nav item → RFP wizard
2. Select up to 3 SAP modules and 3 development types
3. Enter requirements (plain/rich text, screenshots), attachments, and reference ABAP
4. **Save draft** or **submit**

### 4.2 Interview and proposal

After submit: **AI interview** (round/question limits) → **Development Proposal**. View or print (PDF) on the hub **Proposal** tab; **regenerate** consumes AI credits.

### 4.3 Hub phases

| Phase | Content |
|-------|---------|
| **Request** | Body, attachments, reference code |
| **Interview** | Q&A |
| **Proposal** | Proposal text |
| **FS** | Functional spec |
| **Dev code** | Delivered ABAP |
| **As-built** | ZIP or single deliverable file |
| **Settlement** | Fee agreement, payment, payout confirmation |

### 4.4 Offers, matching, inquiries

Consultant **offers**; requester **match**; **inquiries**; **AI inquiry** float (credits).

### 4.5 Dashboard

List, filter, and open hub phases for your RFPs (`/services/abap` landing).


## 5. Analyze & Improvement

1. Use the **Improve** home tile or top-nav item → `/abap-analysis`
2. Submit requirements, existing ABAP, and attachments
3. Review **analysis results** (program/section tabs where shown)
4. Continue on the hub: request, analysis, interview, proposal, FS, code, as-built, offers, **settlement** — same pattern as New ABAP Development where enabled


## 6. SAP-Connected Development

1. Use the **Connect** home tile or top-nav item → `/integration`
2. Submit VBA, Python, API, batch, or similar **non-ABAP / SAP-connected** work
3. Reference ABAP may appear in program/section structure when uploaded
4. Hub phases mirror New ABAP Development: interview → proposal → FS → code → as-built → offers → **settlement**

## 9. Consultant members

### 9.1 Approval and Console

After consultant approval, use **Console** in the top nav (**Request Console**, `/request-console`) to browse other members’ requests (within policy) and submit offers.

### 9.2 AI credits and project settlement

AI features deduct your **AI credits**. On matched requests, complete delivery and **project settlement (payee)** in the hub **Settlement** phase.


## 10. General members · settlement (payer)

On matched requests, use hub **Settlement** to agree amount, fund (bank transfer or online payment), and confirm payout. The Company is not a party to the underlying contract (Terms Article 8).
